Part 1

What is an AI Agent?

An AI agent is a system that perceives its environment, reasons about what it observes, and acts to achieve a goal — then repeats this cycle.

Unlike a simple chatbot that responds to a single question, an agent operates over time in a dynamic environment. It maintains a goal, gathers information through sensors or inputs, uses an AI model to decide what to do next, and executes actions that change the world around it. It may also retain memory of past experience to inform future decisions.

The key insight is the loop: perceive → reason → act → perceive again. This cycle — not any single decision — is what makes a system an agent rather than a mere question-answering tool.

Part 2

About This Demo

The demo shows a concrete AI agent playing Minesweeper — a puzzle game where a minefield must be cleared without triggering any hidden mines.

The agent reads the visible board and sends a structured text description of it to a large language model. The model returns its moves in plain text (e.g. OPEN D4), followed by a brief explanation of its reasoning. The agent parses these instructions and executes them on the board. The full exchange is shown on screen, so you can follow the agent's logic step by step.

After each game, the AI writes a brief reflection in natural language — its "memory" — which is stored in the browser and fed back to it at the start of the next game. This illustrates how agents can accumulate experience over time.

The AI has no memory of its own; continuity is maintained entirely by the agent re-sending everything each time.
Part 3

Key Concepts to Remember

CONCEPTWHAT IT MEANS
Perception–Action Loop The agent observes the environment, decides, acts, and then observes again. This cycle — not any single decision — is what defines agency.
Environment The world the agent operates in. Here it is the Minesweeper board. In real applications it could be the web, a legal database, a contract, or a computer desktop.
System Prompt The instruction document given to the AI before the conversation starts. It defines the agent's role, rules, and output format. Editing it changes behaviour instantly — like rewriting the agent's constitution.
Stateless AI The AI model itself remembers nothing between calls. All context — history, board state, memory — is reconstructed and re-sent by the agent each time. The appearance of continuity is engineered, not intrinsic.
Memory Information retained across episodes. Here the AI writes its own memory in natural language after each game. True long-term memory requires external storage; this demo uses the browser's local storage.
Uncertainty & Guessing When logic is exhausted, the agent must guess. It marks such moves explicitly ("guess"). A well-designed agent does not pretend certainty it does not have.
Grounding The process of connecting the AI's abstract language to concrete actions. Here, "OPEN D4" must be parsed and executed on the actual board. This translation layer is the agent's job.
Multi-model Support The same agent code works with different AI models — Llama, Gemini, Claude, and others. Switching models changes the quality and style of reasoning without changing anything else in the system.
