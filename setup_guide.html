Setup Guide

How to get free API keys and deploy the proxy — step by step

The demo can connect to AI in two ways. The primary way (recommended) uses your own free API keys for Groq or Google Gemini, routed through a small Cloudflare Worker that keeps the keys hidden from the browser. The fallback uses Puter.js, which requires no setup but has a non-renewable credit limit.

This guide covers: getting the keys, deploying the worker, and entering the URL in the demo.

Part A — Get Your Free API Keys

Get one or both. Groq is fastest; Gemini has higher daily limits.

🟣 GROQ — up to 14,400 requests/day, resets daily, no credit card

Go to console.groq.com and click Sign Up. You can sign up with Google or GitHub — no credit card is needed.

Once logged in, click API Keys in the left sidebar, then Create API Key. Give it any name (e.g. "minesweeper-demo").

Copy the key — it starts with gsk_…. Save it now; it will not be shown again. You will paste it into the Cloudflare Worker later.

🔵 GOOGLE GEMINI — generous daily limits, resets daily, no credit card

Go to aistudio.google.com and sign in with your Google account.

Click Get API Key (top left), then Create API key. Select "Create API key in new project" if prompted.

Copy the key — it starts with AIza…. Save it for the next part.

Note: On the free tier, Google may use your prompts to improve its models. This is fine for a demo but worth mentioning to your students.

Part B — Deploy the Cloudflare Worker

Cloudflare Workers are tiny serverless functions that run on Cloudflare's global network. The free tier allows 100,000 requests/day — far more than you need. No credit card is required.

Go to dash.cloudflare.com/sign-up/workers-and-pages and create a free Cloudflare account (email + password, no credit card).

After signing in, click Workers & Pages in the left sidebar, then click Create → Create Worker.

Give the worker any name (e.g. ai-proxy). Cloudflare will assign it a URL like https://ai-proxy.your-name.workers.dev. Click Deploy.

You are now in the online code editor. Delete all existing code in the editor and paste the entire contents of the worker.js file provided with this demo.

Tip: The worker.js file is included in the same folder as this guide. Open it in any text editor, select all, copy, and paste into the Cloudflare editor.

Click Deploy (top right). The worker is now live, but has no API keys yet.

To add your API keys, click the Settings tab (while viewing your worker), then Variables and Secrets, then Add variable.

Add these variables (use Secret type to keep them hidden):

Variable name:  GROQ_API_KEY
Value:          gsk_…your Groq key…

Variable name:  GEMINI_API_KEY
Value:          AIza…your Gemini key…

You only need to add the keys you actually obtained. Click Save and deploy after adding each one.

Copy your worker URL — it looks like https://ai-proxy.your-name.workers.dev. You can find it in the worker dashboard under Settings → Domains & Routes.

Part C — Connect the Demo

Open the demo page. At the top, the Groq / Gemini (Proxy) tier should already be selected.

Paste your Cloudflare Worker URL into the Worker URL field.

Select a model — Groq models (purple dots) use your Groq key; Gemini models (blue dots) use your Gemini key.

Click New Game, then AI Step. The connection indicator will turn green on the first successful call.

If something goes wrong: open the browser console (F12 → Console tab). Error messages from the worker will appear there. The most common issue is a missing or incorrect worker URL.

Part D — Switching to Puter.js (Fallback)

If the proxy is not working or you want to quickly demo with Claude, click the Puter.js (Fallback) button in the connection bar. This requires a Puter account (free signup at puter.com). Note that Puter's AI credits are non-renewable on the free tier.

Summary: What Goes Where

API keys (secret)    → Cloudflare Worker environment variables
                         (never visible in browser source code)

Worker URL (public)  → pasted into the demo page's Worker URL field
                         (safe to share — it only accepts POST requests)

Demo page            → calls your Worker URL
                         Worker forwards to Groq or Gemini
                         Response returns to demo
